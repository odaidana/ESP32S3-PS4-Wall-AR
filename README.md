# ESP32-PS4-Host-S3

سيرفر ويب محلي ومستقل بالكامل يعمل على شريحة ESP32-S3 (N16R8) لاستضافة وتشغيل جيلبريك البلايستيشن 4 (GoldHEN) بدون الحاجة لاتصال إنترنت.

## الهدف من المشروع
الاعتماد على سيرفرات الـ DNS العامة أو المواقع الخارجية لتشغيل جيلبريك الـ PS4 قد يكون غير مستقر، أو يعرض الجهاز لخطر التحديث الإجباري من سوني. هذا المشروع يحول بوردة ESP32-S3 إلى نقطة اتصال محمولة ومغلقة تماماً. بمجرد شبك البلايستيشن عليها، يتم حجب سيرفرات سوني نهائياً وتفعيل التعديلة بسرعة عالية من الذاكرة المحلية للبوردة.

## المكونات المطلوبة
- لوحة تطوير ESP32-S3 (نسخة N16R8).
- كابل USB-C يدعم نقل البيانات (للبرمجة والتغذية).

## آلية العمل
1. **نقطة اتصال Wi-Fi:** تقوم البوردة ببث شبكة لاسلكية خاصة باسم `PS4-Jailbreak-Host`.
2. **توجيه الـ DNS:** أي طلب تصفح يخرج من الـ PS4 (سواء من دليل المستخدم أو المتصفح) يتم تحويله تلقائياً إلى السيرفر الداخلي للبوردة.
3. **إرسال الـ Payload:** يتم استغلال مساحة الـ Flash الضخمة (16MB) لتخزين ملفات الـ HTML والـ JavaScript الخاصة بالثغرة، وتمرير كود الـ GoldHEN مباشرة إلى ذاكرة الكونسول.

## طريقة التركيب والتشغيل
1. افتح كود المشروع باستخدام بيئة **Arduino IDE**.
2. تأكد من تثبيت حزمة تعريفات ESP32 (الإصدار 3 فما فوق).
3. اختر تقسيم الذاكرة المناسب (Partition Scheme) ليتوافق مع حجم ملفات الثغرة (مثل `16MB Flash with SPIFFS/LittleFS`).
4. ارفع الكود إلى الـ ESP32-S3.
5. ارفع ملفات الجيلبريك إلى نظام الملفات الداخلي للبوردة باستخدام أداة `Sketch Data Upload`.
6. اشبك الـ PS4 على شبكة الواي فاي الجديدة وافتح دليل المستخدم (User's Guide) لتفعيل التعديلة.
